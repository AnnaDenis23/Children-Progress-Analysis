# Children Progress Analysis 

## Описание проекта

Этот проект анализирует данные по занятиям детей и автоматически выявляет случаи отсутствия прогресса (stagnation) по различным навыкам (domain).

Основная задача — найти детей, у которых:

* долго не наблюдается улучшение результатов
* есть риск застоя в развитии
* наблюдается слабая динамика после длительного плато

---



## Данные

Входной файл:

```
data/children_sessions.xlsx
```

Содержит информацию о занятиях:

* `child_id` — идентификатор ребенка
* `age` — возраст
* `diagnosis` — диагноз
* `domain` — навык (например, Listening, Social и т.д.)
* `session_date` — дата занятия
* `assessment_score` — оценка прогресса
* `comment` — комментарий специалиста
* `progress_flag` — исходный флаг прогресса (не всегда корректный)
* `specialist_type` — тип специалиста

---

## Проблемы в данных

В исходных данных были обнаружены ошибки:

* `progress_flag` иногда содержит тип специалиста
* `specialist_type` иногда пустой
* значения флагов могут быть на русском (`импровед`, `стагнант`)
* есть пропуски и некорректные значения

В проекте реализована автоматическая очистка данных:

* исправление опечаток (`импровед` → `improved`)
* перенос специалиста в правильную колонку
* заполнение пропусков (`unknown`)

---

## Логика анализа

### 1. Автоматический progress_flag

Так как исходный `progress_flag` ненадежен, он пересчитывается:

* score вырос → `improved`
* score не изменился или снизился → `stagnant`
* первая запись → `unknown`

---

###  Основное правило stagnation

Для каждого ребенка и навыка:

* определяется дата последнего улучшения
* считается количество дней без прогресса

Если:

```
days_without_progress >= 28
```

→ случай считается stagnation

---

###  Анализ последних 28 дней

Дополнительно анализируются последние 28 дней от последней сессии:

* количество занятий
* значения score
* был ли прогресс в этом периоде

Добавленные поля:

* `sessions_last_28_days`
* `scores_last_28_days`
* `progress_in_last_28_days`


## Как запустить

### 1. Создать виртуальное окружение

```bash
python3 -m venv venv
```

### 2. Активировать

```bash
source venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Запустить анализ

```bash
python -m src.cli
```

## Ограничения

* используется простая логика оценки прогресса (по score)
* не учитывается сложность заданий
* не анализируется текст комментариев (NLP не применялся)
* интервалы между занятиями могут влиять на результат
